Conexión a WhatsAp

El primer punto a tener en cuenta es acceder a la interfaz gráfica del programa, para ello ingresaremos en un navegador en la soguiente ruta: http://localhost:5502. Para la correcta conexión con WhatsAp es necesario tener el dispositivo a utilizar desvinculado de cualquier otra sesión previa al bot. Luego, en la página de inicio del bot, presionamos el botón "Conectarse a WhatsAp", esperamos que se abra la ventana que contiene el código QR que escanearemos para establecer la conexión. Una vez escaneado deberemos esperar la sincronización de WhatsApp, en este caso, en el dispositivo conectado deberá aparecer la leyenda "Sesión Activa". Esto significa que la conexión se realizó de manera satisfactoria.

Carga de Base de Datos

La carga de la base de datos requiere de varios puntos a tener en cuenta para evitar cualquier error a la hora del envío de mensajes. A continuación se detallan:
1. Tipo de Archivo:

Sólo se acepta archivo Excel .xls/.xlsx

2. Formato de Columnas:

Se deben tener en cuenta los espacios en los títulos de las columnas, no debe existir ninguno, y, los nombres deben estar en mayúscula.

3. Formato de Celdas:

Cuit: No debe tener guiones, espacios ni puntos.
Número de contacto: debe respetar la siguiente forma: 5493510000000, es decir, código de país, código de área sin el "0" y el número de teléfono sin "15".
Celdas sin Datos: No deben existir celdas vacías, siempre tienen que tener un dato, para informar ausencia de datos se puede escribir "-".

Actualización de base de Datos

Para la actualización de la base de datos, sólo se debe subir la tabla con los valores a modificar, no es necesario subir toda la tabla. En el caso de agregar nuevos valores, también sólo hay que subir una nueva tabla con dichos valores nuevos.

Envío de Vep

En este caso el procedimiento es sencillo ya que con tocar el botón de envío de Vep el programa se encarga de hacer la búsqueda de los clientes que tienen como atributo la columna de "VEP" en "SI". En lo que se debe prestar atención, es en el momento en el que se guarda el archivo. Para nombrarlo, solamente tenemos que utilizar su número de CUIT tal y como se lo utiliza en la tabla de la base de datos.

Envío de Mensaje y Recordatorio

Para el envío de Mensajes y Recordatorios el procedimiento es el mismo, se utiliza el botón correspondiente a la acción que se desea realizar teniendo en cuenta que en la base de datos se deja previsto en dos columnas diferentes si al cliente hay que enviarle el mensaje o el recordatorio. Por este motivo son dos botones diferentes, cada uno verifica en la base de datos que el cliente esté habilitado para recibir uno u otro. Esto es importante tenerlo en cuenta a la hora de realizar las operaciones para no tener errores involuntarios, los cuales, no son verificados por el sistema. Por ejemplo si se quiso enviar un recordatorio a un cliente que lo tiene inhabilitado en la base de datos, el sistema no informará un error sino que no enviará el recordatorio a ese cliente, lo mismo ocurriría con la operación de mensajes.
